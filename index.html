1.1 - Substituição de valores nulos pela Média

Quando lidamos com atributos que possuem dados contínuos, a abordagem mais adequada para substituir valores ausentes é utilizando a média. No nosso conjunto de dados, a variável "idade" é contínua, representando valores como 18, 21, 19, e assim por diante. Se alguns desses valores estiverem ausentes, podemos preenchê-los com a média da variável.

Dados contínuos referem-se a variáveis que podem assumir qualquer valor dentro de um intervalo específico, como a idade, que pode ser representada por números reais. Em contraste com dados discretos, que assumem valores específicos e geralmente são inteiros, dados contínuos podem incluir frações ou números decimais.

Utilizando o código abaixo, substituímos os valores ausentes na coluna "idade" pela média dessa mesma coluna:

media_idade = df['idade'].astype('float').mean(axis=0)

A estratégia de substituição pela média é adotada para manter a coerência na distribuição dos dados.

1.2 - Substituição de valores nulos pelo valor mais Frequente

Exemplificativo¶

Quando lidamos com atributos que contêm dados categóricos, a abordagem recomendada para substituir valores ausentes é utilizar o valor mais frequente na categoria. Vamos considerar o cenário em que temos uma coluna categórica, por exemplo, a coluna "estado_civil", e alguns valores estão ausentes. A estratégia apropriada seria substituir esses valores pelo estado civil mais frequente.

Dados categóricos referem-se a variáveis que representam categorias discretas, como "Solteiro", "Casado" ou "Divorciado". Ao depararmos com valores ausentes nesse tipo de atributo, é benéfico preenchê-los com o valor mais comum da categoria, mantendo a integridade da distribuição.

A seguir, apresento um código genérico que exemplifica como substituir, com o valor mais frequente, os valores ausentes na coluna categórica:

valor_mais_frequente_estado_civil = df['estado_civil'].value_counts().idxmax()
df['estado_civil'].replace(np.nan, valor_mais_frequente_estado_civil, inplace=True)

Essa abordagem garante que os valores ausentes sejam preenchidos de maneira representativa, isso visa manter a coesão na distribuição dos dados categóricos.

Formatos Variados: Datas podem ser apresentadas em diversos formatos, como "dia/mês/ano" ou "ano/mês/dia", e horários podem incluir ou não segundos. A variedade de representações adiciona complexidade ao processo de conversão.

Presença de Texto Adicional: Às vezes, datas e horas podem estar "misturadas" a outros dados de texto, tornando necessário separar e converter essas informações de maneira cuidadosa.

Precisão e Granularidade: Dependendo do contexto, pode ser necessário trabalhar com diferentes níveis de precisão temporal, como dias, horas, minutos ou até mesmo segundos. A manipulação precisa desses níveis de granularidade adiciona camadas de complexidade.

2.3.1 Coluna CGPA: Lidando com valores intervalares¶

Ao explorar a coluna CGPA, notamos que os dados estão apresentados de forma métrica, representando intervalos como '3.00 - 3.49', '3.50 - 4.00', etc. Esses intervalos sugerem faixas específicas de Médias Cumulativas de Notas (CGPA), proporcionando uma visão geral do desempenho acadêmico dos alunos.

Agora, para tornar esses intervalos mais significativos numericamente, recorreremos ao uso de expressões regulares (regex). Vamos entender como isso funciona:

Entendendo Expressões Regulares de Forma Simples:¶

Expressões regulares, ou regex, são padrões de caracteres que facilitam a busca e manipulação de textos. No contexto da coluna CGPA, onde os valores estão no formato '3.00 - 3.49', a expressão regular simples \d+\.\d+ nos ajuda a extrair os números ali contidos.

  • \d: representa qualquer dígito de 0 a 9.
  • +: indica que o caractere anterior (no nosso caso, \d) pode aparecer uma ou mais vezes.
  • .: procura por um ponto decimal, precedido por uma barra invertida para ser interpretado literalmente.

Dito isso nossa próxima etapa é converter esses "intervalos textuais" em valores numéricos mais representativos. Entendemos que cada intervalo abrange uma faixa de valores, e para simplificar a análise, assumiremos a média desse intervalo como seu valor representativo. Se não entendeu o que isso quer dizer ainda, buscaremos um valor entre os valores apresentados no intervalo e esse valor será a média entre eles.

Escolhendo a Técnica Certa:¶

Como já vimos, a coluna cgpa tem seus dados em formato textual (string) e precisamos extrair os números ali apresentados. Então, para extrair números de strings, a utilização de expressões regulares é uma escolha comum e poderosa.

Como Utilizar:¶

É útil entender o padrão dos dados que estamos tentando extrair. No nosso caso, o padrão foi apresentado enquanto explicávamos o que são expressões regulares. Existem recursos online e bibliotecas em Python, como a re, que fornecem ajuda na criação de expressões regulares. A intenção não é ensinar a resolver todos os casos possíveis, mas mostrar a forma como dados podem ser apresentados e como isso impacta na escolha do tipo de tratamento a ser dado em determinada situação.

Abaixo criamos uma função chamada calcular_media. Essa função utiliza a biblioteca re (expressões regulares) para extrair todos os números decimais (\d+.\d+) de uma string que representa um intervalo, como '3.00 - 3.49'. A função então converte esses números para o formato float e calcula a média entre eles.

In [14]:
# Importando a biblioteca re, que lida com expressões regulares
import re

# Definindo uma função chamada calcular_media
def calcular_media(intervalo):
    # Utilizando re.findall para encontrar todos os padrões de números decimais na string do intervalo
    numeros_encontrados = re.findall(r'\d+\.\d+', intervalo)
    
    # Inicializando uma lista vazia para armazenar os números convertidos
    numeros_convertidos = []
    
    # Iterando sobre cada número encontrado na lista
    for numero in numeros_encontrados:
        # Convertendo cada número para o tipo de dado float e adicionando à lista numeros_convertidos
        numeros_convertidos.append(float(numero))
    
    # Calculando a média dos números na lista numeros_convertidos
    media = sum(numeros_convertidos) / len(numeros_convertidos)
    
    # Retornando a média calculada
    return media

# Aplicando a função calcular_media à coluna 'cgpa' do DataFrame df
df['cgpa'] = df['cgpa'].apply(calcular_media)

3 - Padronização e Normalização

Depois de explorarmos a identificação e tratamento de dados ausentes, bem como o manejo de tipos de dados, agora vamos focar na padronização e normalização.

Motivação:

A padronização e normalização asseguram que as variáveis estejam em escalas comparáveis, eliminando potenciais viéses em algoritmos sensíveis à escala dos dados.

Implicações:

  • A ausência desses processos pode levar a resultados distorcidos, especialmente em algoritmos que dependem de medidas de distância ou otimização.
  • A padronização contribui para uma interpretação mais intuitiva dos coeficientes ou pesos do modelo, facilitando a análise e compreensão dos resultados.

Sigamos com um exemplo:

Vamos imaginar as alturas de duas pessoas, João e Maria, para entender como a falta de uniformidade nas escalas pode afetar a interpretação dos dados.

João mede 180 centímetros, enquanto Maria tem 160 centímetros. Se tentarmos comparar diretamente esses números, pode parecer que João é muito mais alto que Maria. No entanto, essa conclusão poderia ser enganosa se não considerarmos a variação natural nas alturas médias entre diferentes grupos de pessoas. Cada pessoa é única, e as características físicas podem variar independentemente de gênero.

Imagine que estamos utilizando essas alturas em um conjunto de dados, como se fosse um grande quebra-cabeça com muitas peças diferentes. Sem padronização ou normalização, podemos enfrentar um desafio semelhante a tentar resolver o quebra-cabeça usando apenas uma cor, como se a altura fosse a única peça importante.

Sem esses processos de ajuste (normalização e padronização), nossas "regras" (ou algoritmos) podem ficar um pouco desequilibradas, dando mais importância à altura do que a outras características igualmente relevantes; como por exemplo características biológicas específicas. É como se estivéssemos julgando a habilidade de alguém apenas com base na altura, ignorando completamente suas outras características únicas.

A padronização e normalização desempenham um papel relevante e essencial na comparação de atributos, garantindo que todos contribuam de maneira equitativa para as análises. A ausência desses processos pode levar a conclusões enviesadas, já que algumas variáveis podem exercer influência desproporcional devido às suas escalas originais.

As técnicas de padronização e normalização não apenas facilitam a interpretação dos resultados, mas também melhoram a convergência de algoritmos, tornando a análise mais precisa e significativa.

Dessa forma, nossa abordagem não apenas trata a ausência de dados, mas também assegura que, ao comparar diferentes características, estejamos considerando suas contribuições de maneira justa e objetiva.

Utilizamos a normalização para comparar grandezas que apresentam diferentes magnitudes ou escalas. Essa prática que visa colocar as variáveis em uma escala comum, garantindo que nenhuma delas exerça influência desproporcional nas análises.

Ao explorar nosso conjunto de dados, percebemos que temos variáveis como idade, peso_kg, e altura_cm que apresentam diferentes magnitudes. A normalização é uma técnica útil nesse contexto, pois nos permite comparar e interpretar essas variáveis de maneira mais intuitiva e justa.

Suponhamos que temos os seguintes dados para três alunos:

A 18 70 180
B 22 65 160
C 19 75 170

Ao observar diretamente esses valores, pode ser difícil comparar a influência relativa de cada variável devido às suas diferentes escalas. A idade, por exemplo, tem valores na faixa de dezenas, enquanto o peso e a altura estão na faixa de unidades e centenas, respectivamente.

Sem Normalização:

  • Comparar diretamente a idade de A (18) com a altura de B (160) não fornece uma compreensão clara da diferença real entre essas variáveis.

Normalização: A normalização nos permite transformar essas variáveis em uma escala comparável, facilitando a interpretação e a comparação. Vamos normalizar a idade, peso_kg e altura_cm usando a fórmula:

Valor Normalizado = Valor original - Valor mínimo / Valor máximo - Valor mínimo

  • Agora, podemos comparar intuitivamente as três variáveis normalizadas para cada aluno. A idade normalizada varia entre 0 e 1, assim como o peso e a altura, permitindo uma comparação "mais justa".

Ao adotar a normalização, não apenas facilitamos a interpretação; mas também tornamos os dados mais acessíveis visualmente, especialmente quando diferentes variáveis estão envolvidas. Essa prática contribui para análises mais transparentes e interpretações mais precisas.

4 - Visualização com Gráficos de Barras Agrupadas (Binning)

Agora observaremos uma etapa que visa simplificar a representação de variáveis contínuas. Em vez de lidar com valores individuais, o binning organiza esses valores em intervalos discretos ou "bins". Esta abordagem facilita a interpretação e visualização dos dados, permitindo a identificação de padrões e tendências de maneira mais clara.

Motivação:

Implicações:

7.1 - Compreendendo a Semente de Aleatoriedade (Seed)

Em ciência de dados, pode-se fazer necessária a geração de números aleatórios, no entanto, para garantir consistência nos resultados entre diferentes execuções de um código, fazemos uso da semente de aleatoriedade (seed).

Analogia com Caixas e Bolas:

Vamos imaginar que temos várias caixas, cada uma contendo uma fila única de bolas coloridas. Cada vez que retiramos uma bola, a ordem é determinada pela própria caixa. Aqui, as caixas representam diferentes sequências pseudoaleatórias.

A semente de aleatoriedade é como escolher uma caixa específica para começar. Se escolhermos a mesma caixa (mesma semente) toda vez, retiraremos as bolas na mesma ordem, como se elas estivessem pacientemente alinhadas na fila, esperando para serem retiradas.

Exemplo Prático:

Consideremos duas caixas:

  • Caixa 1 (Semente = 1): Bolas azul, verde, roxa.
  • Caixa 2 (Semente = 2): Bolas roxa, azul, verde.

Se optarmos pela caixa 1 (seed = 1) np.random.seed(1), retiraremos as bolas na sequência azul, verde, roxa. Se outra pessoa escolher a mesma caixa 1, ela também retirará as bolas na mesma ordem predefinida. De maneira análoga, se optarmos pela caixa 2 (seed = 2) np.random.seed(2), a sequência será roxa, azul, verde, e qualquer pessoa que escolher a caixa 2 retirará a mesma sequência predefinida.

A semente de aleatoriedade (np.random.seed(1)) proporciona consistência, como se as bolas estivessem sempre dispostas em uma fila, aguardando serem retiradas na mesma ordem, independentemente de quem as estiver retirando.

Essa abordagem assegura a reprodutibilidade dos resultados, ou seja, não importa quem for "executar" a tabela que criamos, os resultados obtidos serão sempre os mesmos; o que nos proporciona uma base sólida para análises comparáveis e reproduzíveis.

7.4 Mais algumas palavras sobre Normalização

A normalização é frequentemente utilizada ao preparar dados para algoritmos de aprendizado de máquina. Aqui estão algumas razões pelas quais esse processo é enfatizado:

  • Escalas Comparáveis: Algoritmos que utilizam medidas de distância, podem ser influenciados pela escala dos dados. A normalização coloca todas as variáveis em uma escala comparável, evitando que uma variável com uma escala maior domine a influência nos resultados.

  • Convergência mais Rápida: Algoritmos de otimização, como aqueles usados em redes neurais, geralmente convergem mais rápido em dados normalizados. Isso pode levar a um treinamento mais eficiente do modelo.

  • Influência do Outlier Mitigada: A normalização ajuda a reduzir a influência de valores discrepantes (outliers) ao colocar todos os dados em uma escala mais uniforme.

  • Interpretação Mais Intuitiva: Quando os dados estão normalizados, a interpretação dos coeficientes ou pesos do modelo torna-se mais intuitiva, pois as variáveis estão em escalas comparáveis.

  • Preparação de Dados para Algoritmos Específicos: Alguns algoritmos, como redes neurais, são sensíveis à escala dos dados. Normalizar os dados ajuda a criar uma entrada mais apropriada para esses algoritmos.

7.5 Considerações sobre Binning

Antes de realizar o agrupamento (binning), precisamos determinar a faixa intervalar apropriada para a variável a ser analisada. A escolha da faixa intervalar afeta diretamente a interpretação dos dados e a eficácia do agrupamento.

Aqui estão algumas considerações ao determinar a faixa intervalar:

  • Entendimento do Domínio: É necessário conhecer o contexto do problema e entender a natureza da variável. Por exemplo, ao agrupar idades, pode ser interessante escolher intervalos significativos, como faixas etárias comuns (por exemplo, 0-18, 19-35, 36-50, etc.).

  • Distribuição dos Dados: Se os dados estão concentrados em uma faixa estreita, pode ser útil escolher intervalos menores para capturar detalhes específicos. Se os dados são mais dispersos, intervalos maiores podem ser mais apropriados.

  • Objetivos Analíticos: Considere os objetivos específicos da análise. Às vezes, intervalos mais amplos são escolhidos para fornecer uma visão geral, enquanto intervalos menores podem ser necessários para análises mais detalhadas.

  • Relevância para o Público-alvo: Pense no público-alvo da análise. Intervalos significativos e compreensíveis para o público tornam a interpretação mais acessível.

Antes de implementar o agrupamento, realizar uma análise exploratória dos dados pode ajudar a informar a escolha da faixa intervalar mais apropriada, no entanto, esse aspecto não está contido no objetivo desse projeto.
